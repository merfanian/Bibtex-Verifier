How it works

1

Upload

Drop your .bib file above or click to browse. The file is read entirely in your browser -- nothing is uploaded to a server.

2

Verify

Each entry is looked up by title against Semantic Scholar and CrossRef. The tool compares authors, year, venue, DOI, and other fields to the published record.

3

Review

Results are color-coded: Verified means everything matches, Auto-Updated shows a diff of changed fields, and Needs Review means the title didn't match closely enough. Accept or reject each suggestion.

4

Download

Click the download button to get a corrected .bib file with your accepted changes applied. Ready to use in your LaTeX project.

FAQ

Is my data safe?

Yes. Your .bib file is parsed entirely in your browser using JavaScript. The only network requests made are title lookups to the public CrossRef and Semantic Scholar APIs. No file data is sent to any server.

Why are some entries "Not Found"?

Not every paper is indexed in CrossRef or Semantic Scholar. ArXiv preprints, workshop papers, theses, and very recent publications may not be found. The tool will flag these so you can verify them manually.

Why does verification take a while?

To respect the free APIs' rate limits, the tool sends requests at a controlled pace (about 1 per second to Semantic Scholar). For a file with 50 entries, expect roughly 2-3 minutes.

Can I use this offline?

The tool requires an internet connection to query the academic APIs. For offline use, you can run the Python backend version locally via Docker.

Is this open source?

Yes! The full source code is on GitHub under the MIT license. Contributions are welcome.
